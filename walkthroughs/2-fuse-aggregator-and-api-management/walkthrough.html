Managing and protecting APIs using API management

This Solution Pattern demonstrates how an API can be protected and rate limited.

Organizations modernizing their applications typically create many APIs. It then becomes paramount to understand the usage of these APIs and to ensure they are protected, for example, from a denial-of-service (DoS) attack. It is possible to protect and limit access to APIs as part of an integration.

In this walkthrough, you use the Fuse Aggregation App, which is a demonstration app that provides data using an API. The Solution Pattern shows how to deploy, edit, protect and rate-limit that API, using OpenShift, Launcher, CodeReady Workspaces and 3Scale.

Running the Fuse Aggregation App

Deploying the Fuse Aggregation App

The Fuse Aggregation App provides information about flights. You deploy it using Launcher.

  1. Log in to the Launcher wizard.

  2. Select the option to Build and Deploy Online in the Target Environment step.

  3. Click the Authorize button to allow Launcher create resources in the OpenShift cluster. This triggers an OAuth flow and eventually redirects back to the Launcher wizard.

  4. Click the blue down-arrow icon to continue.

  5. Select the Aggregation mission and Fuse runtime in the Mission & Runtime step

  6. Click the blue down-arrow icon to continue.

  7. Authorize your Github Account

    1. Click the Log In & Authorize Account button in the Authorize Git Provider step.

    2. This opens a Github OAuth page.

    3. Click the Authorize <organisation/user> button at the bottom.

    4. You will be redirected back to the Launcher wizard.

  8. Click all the blue down-arrow icons to continue.

  9. Click the Set Up Application button to create a Git repository and provision the mission resources to OpenShift. A build for the Fuse Aggregation App is triggered and deployed.

Check that the following pods are running in the OpenShift Project:

  • fuse-flights-aggregator

  • arrivals

  • departures

Modifying the Fuse Aggregation App

Modify the Fuse Aggregation App to aggregate flights data from the Arrivals & Deparatures servers

  1. Log in to the CodeReady Workspaces Dashboard

  2. The New Workspace page is displayed. Create a new Workspace:

    1. Use the Java 1.8 stack.

    2. Click the Add or Import Project button.

    3. Click the Github tab.

    4. Click the Connect your Github Account button.

    5. Select the checkbox beside the git repository named {walkthrough-namespace}. You may need to filter by account if you are a member of any Organizations.

    6. Click the Add button to add the repository to the workspace.

    7. Click the Create & Open button at the bottom.

    8. Click Open in IDE.

  3. Open the file at src/main/java/com/redhat/fuse/boosters/rest/http/CamelRouter.java.

  4. Comment out the routing code that talks to local java services.

    1. Navigate to the section of the file with a comment of // COMMENT OUT THIS.

    2. Comment out the line of code below this using double slashes //.

  5. Uncomment the routing code that talks to remote services.

    1. Navigate to the section of the file with a comment of // UNCOMMENT THIS.

    2. Uncomment the line of code below this by removing the double slashes.

  6. Commit and push the changes back to the repository:

    1. Select the Git menu, then Commit.

    2. Ensure the CamelRouter.java file is checked.

    3. Enter a commit message of Switch to remote services in the input area.

    4. Check the box for Push commited changes to and ensure the branch is set to master.

    5. Click the Commit button.

    6. A green notification Pushed to Origin is displayed.

    7. A new build is triggered in OpenShift and the new changes are rolled out to the Fuse Aggregation App.

After waiting for the build and deployment to complete, check Flights Endpoint. Are more than 8 flights displayed?

Verify that you followed each step in the procedure above. If you are still having issues, contact your administrator.

API Management Login

  1. Open the Red Hat 3scale Login screen.

  2. Click the Red Hat Single Sign On option. This triggers an OAuth Flow and redirects you back to the Red Hat 3scale Dashboard.

Can you see the Red Hat 3scale Dashboard and navigate the main menu?

Verify that you followed each step in the procedure above. If you are still having issues, contact your administrator.

Adding the Fuse Aggregation App Endpoint to Red Hat 3scale

  1. From the Dashboard, select the New API item.

  2. Select the Define Manually option.

  3. Enter the following as the Name and System name:

    fuse-flights-aggregator-{user-sanitized-username}

  4. Leave the Description field empty.

  5. Click Add API at the bottom of the screen.

  6. From the Overview screen, select the Configure APIcast button.

  7. In the Private Base URL field, enter:

    {route-fuse-flights-aggregator-host}

  8. In the Staging Public Base URL, enter:

    https://wt2-{user-sanitized-username}-3scale.{openshift-app-host}

  9. Click Update & test in Staging Environment to save your work.

    You might encounter a 403: Authentication failed or 503 Service Unavailable message. You can ignore these messages, the issue is resolved in a later step.

Setting Fuse Aggregation App Endpoint Limits

  1. Create a new Application Plan:

    1. Click Applications > Application Plans from the side navigation.

    2. Click Create Application Plan.

    3. Enter the following for Name and System name:

      fuse-flights-aggregator-{user-sanitized-username}

    4. Leave the other fields with their default values.

    5. Select Create Application Plan. You will be redirected to the Application Plans screen.

    6. Select the Publish button, beside your plan list item, to publish the Plan.

  2. Select the fuse-flights-aggregator-{user-sanitized-username} plan in the list to return to the edit screen.

  3. Set a limit of 5 calls per hour:

    1. From the Metrics, Methods, Limits & Pricing Rules section, click the Limits (0) button.

    2. Click the New usage limit button.

    3. Set the Period to hour.

    4. Set the Max. value to 5.

    5. Click Create usage limit.

    6. Click Update Application plan

  4. Create a new Application for the Developer Group, assigned to the Plan:

    1. Select Audience from the top navigation bar dropdown.

    2. Select the Developer Account to open the Account Summary page.

    3. Select the (num) Application item from the breadcrumb to view Applications.

    4. Click the Create Application button in the top right.

    5. Select the fuse-flights-aggregator-{user-sanitized-username} Plan in the Application plan dropdown.

    6. Enter the following for Name and Description:

      fuse-flights-aggregator-{user-sanitized-username}

    7. Click Create Application.

  5. Set a custom User Key for the application:

    1. On the fuse-flights-aggregator-{user-sanitized-username} application screen you were redirected to, scroll to the API Credentials section.

    2. Click the green pencil icon beside the API User Key

    3. In the Set Custom User Key modal dialog, enter:

      fuse-flights-aggregator-{user-sanitized-username}

    4. Click Set Custom Key.

  1. Select the Overview menu item in the side navigation.

  2. Scroll down and click the Configure APIcast button in the bottom right.

  3. Click the Update & test in Staging Environment button at the bottom again.

  4. Is a success message displayed and a green line along the left side of the page visible?

Verify that you followed each step in the procedure above. If you are still having issues, contact your administrator.

Create a new ActiveDocs Service

  1. Click Active Docs from the side navigation.

  2. Click Create your first spec

  3. Enter the following for Name and System name:

    fuse-flights-aggregator-{user-sanitized-username}

  4. Enter the below content for the API JSON Spec.

    {
  "swagger" : "2.0",
  "info" : {
    "version" : "1.0",
    "title" : "Airport Flights REST API"
  },
  "host" : "wt2-{user-sanitized-username}-3scale.{openshift-app-host}",
  "basePath" : "/camel/",
  "tags" : [ {
    "name" : "flights",
    "description" : "List all flights (arrivals & departures)"
  } ],
  "schemes" : [ "https" ],
  "paths" : {
    "/flights" : {
      "get" : {
        "tags" : [ "flights" ],
        "operationId" : "flights-api",
        "parameters" : [ {
          "name" : "user_key",
          "in" : "query",
          "description" : "User Key, if calling the API in front of 3Scale.",
          "required" : false,
          "type" : "string",
          "x-data-threescale-name": "user_keys"
        } ],
        "responses" : {
          "200" : {
            "description" : "Output type",
            "schema" : {
              "type" : "string",
              "format" : "com.redhat.fuse.boosters.rest.http.FlightsList"
            }
          }
        }
      }
    }
  },
  "definitions" : {
    "Flight" : {
      "type" : "object",
      "properties" : {
        "code" : {
          "type" : "string"
        },
        "time" : {
          "type" : "integer",
          "format" : "int64"
        },
        "flightType" : {
          "type" : "string"
        }
      }
    }
  }
}

  5. Click the Create Service button.
