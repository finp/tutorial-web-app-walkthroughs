Integrating message-oriented middleware with a RESTful API using AMQ Online

This Solution Pattern demonstrates a fault tolerant integration pattern that integrates a message driven architecture with a RESTful API architecture.

Message-oriented middleware (MOM)

MOM allows you to implement an asynchronous, routable, and transformable message queue.

API-centric architecture

Typically, a developer working on a back-end application implements a synchronous RESTful API to allow communication from other applications.

You may identify business value, as part of an application modernization effort, by integrating a message queue with an API. This is not trivial, because of the conflicting underlying timing models. The data may require transformation to be usable.

In this Solution Pattern, create a simple and intuitive integration with built-in monitoring, to enable communication between two applications:

  • A Node.js webapp provides a method to create messages, in this example, a new inventory item.

  • A Spring Boot CRUD app provides a RESTful API, in this example, an inventory API.

To create this integration, use AMQ Online and API connectors in Red Hat Fuse Online.

All of the applications and services used in this Solution Pattern are running on Red Hat OpenShift.

Creating an address in AMQ Online address space

To route messages to the Spring Boot app, create an address in AMQ Online.

  1. Log into AMQ Online.

  2. Select Addresses from the left hand menu.

  3. Create a requests address:

    1. Select the Create button to start the Create new address wizard.

    2. Enter the following as the address name:

      work-queue-requests

    3. Select queue as the type and click Next.

    4. Select Small Queue as the address plan and click Next.

    5. Select Create to complete the wizard.

  4. Monitor the AMQ Online console until a green check mark is displayed beside the new address. Creating an address may take some time.

Is the newly created address displayed on the Addresses screen of the AMQ Online console?

Verify that you followed each step in the procedure above. If you are still having issues, contact your administrator.

Creating connections

Creating an API Connector

Fuse Online is an enterprise integration platform that provides connectors for many services, such as Gmail and IRC. It can be extended with custom connectors, using either OpenAPI definitions (Swagger) or JAR files. To create a connector for the RESTful API, register the OpenAPI definition as a Customization.

  1. Log in to the Red Hat Fuse Online console.

    Note
    		You are not logging into the shared Fuse Online instance, available from the Applications menu with this step. Starting this Solution Pattern provisioned an instance of Fuse Online which is not shared with other cluster users.

  2. Select Customizations > API Client Connectors from the left hand menu.

  3. Select the Create API Connector button to start the API Client Connector wizard.

  4. When prompted to Upload OpenAPI Document, select Use a URL:

    1. Enter the following in the URL field:

      {route-crud-host}/v2/api-docs

    2. Click Next.

  5. When prompted with Review Actions, select Next.

  6. When prompted with Specify Security, select Next.

  7. When prompted with Review/Edit Connector Details:

    1. Enter the following in the Connector Name field:

      Walkthrough 1A CRUD Connector

    2. Click Save.

Is the new connector displayed on the Customizations screen of the Red Hat Fuse Online console?

Verify that you followed each step in the procedure above. If you are still having issues, contact your administrator.

Creating HTTP connection to CRUD App

To enable Fuse Online to send messages from the queue to the Spring Boot CRUD app, create a connection in Red Hat Fuse Online using the API connector you just created.

  1. Log in to the Red Hat Fuse Online console.

  2. Select Connections from the left hand menu.

  3. Select the Create Connection button to start the Create Connection wizard.

  4. When prompted with Select Connector, select Walkthrough 1A CRUD Connector.

  5. When prompted with Configure connection:

    1. Enter the following in the Host field:

      {route-crud-host}

    2. Accept the default value in the Base path field

  6. When prompted with Name connection:

    1. Enter the following in the Name field:

      Walkthrough 1A CRUD Connection

    2. Click Save.

Is the new connection displayed on the Connections screen of the Red Hat Fuse Online console?

Verify that you followed each step in the procedure above. If you are still having issues, contact your administrator.

Creating AMQP connection in Red Hat Fuse Online

To allow Fuse Online to consume messages placed on the queue by the Node.js webapp, create a connection in Red Hat Fuse Online.

  1. Log in to the Red Hat Fuse Online console.

  2. Select Connections from the left hand menu.

  3. Select the Create Connection button to start the Create Connection wizard.

  4. Select AMQP Message Broker to configure an AMQP connection.

  5. Enter the connection URI relating to AMQ Online:

    amqp://{enmasse-broker-url}:5672?amqp.saslMechanisms=PLAIN

  6. Enter the username relating to AMQ Online:

    {enmasse-credentials-username}

  7. Enter the password relating to AMQ Online:

    {enmasse-credentials-password}

  8. Set the value of Check Certificates to Disable.

  9. Select the Validate button to check that the values are valid.

  10. Click Next and enter a name for the connection, for example:

    Walkthrough 1A Messaging App

  11. Click Save.

Is the new connection displayed on the Connections screen of the Red Hat Fuse Online console?

Verify that you followed each step in the procedure above. If you are still having issues, contact your administrator.

Creating an integration

To use the Fuse connections, create an integration in Red Hat Fuse Online.

  1. Log in to the Red Hat Fuse Online console.

  2. Select Integrations from the left hand menu.

  3. Select the Create Integration button to start the Create Integration wizard.

  4. Choose Walkthrough 1A Messaging App as the Start Connection.

  5. When prompted to Choose an Action, select Subscribe for messages.

  6. When prompted for a Destination Name, enter:

    work-queue-requests

  7. Choose Queue as the Destination Type and click Next.

  8. When prompted to Specify Output Data Type:

    1. Select JSON Schema as the type.

    2. Enter the following in the Definition field:

      {
	"$schema": "http://json-schema.org/draft-04/schema#",
	"type": "object",
	"properties": {
		"type": {
			"type": "string"
		}
	}
}

    3. Click Next.

  9. Choose Walkthrough 1A CRUD Connection as the Finish Connection.

  10. When prompted to Choose an Action, select Create a fruit.

  11. When prompted with Configure the action click Next.

  12. When prompted to Add to Integration, click on the blue + icon in the left panel.

  13. Select Data Mapper to map source and target fields in the corresponding JSON schemas:

    1. Expand the body item in the Target tree to reveal the name item.

    2. Click and drag type from the Source column to body/name in the Target column.

    3. Click Done to navigate back to the Integration screen.

  14. Click Publish.

  15. When prompted, enter a name, for example:

    Walkthrough 1A

  16. Click Save and publish.

  17. Monitor the Integration Summary dashboard until a green check mark is displayed beside the new integration. The integration may take some time to complete building.

Is the integration displayed as Running on the Integration screen of the Red Hat Fuse Online console?

  1. Wait for the integration to appear. This can take several minutes.

  2. Verify that you followed each step in the procedure above. If you are still having issues, contact your administrator.

Using the application integration

After setting up the integration between the Node.js and Spring Boot applications, use the integration to create a new fruit for the grocery inventory application.

  1. Navigate to the Node.js webapp.

  2. Enter a value for Fruit, for example:

    Pineapple

  3. Click Send Request.

  4. Navigate to the Spring Boot app.

  5. Check that the entry from the Node.js webapp is displayed.

View the activity log:

  1. Log in to the Red Hat Fuse Online console.

  2. Select Integrations from the left hand menu.

  3. Select your integration.

  4. Click the Activity tab.

  5. Expand the log entry to display the steps performed.

Is your activity displayed?
