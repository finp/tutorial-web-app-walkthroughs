Low-code API development and SaaS integration

This Solution Pattern demonstrates how to implement a container-native OpenAPI RESTful API without coding.

Container-native applications

Applications that are designed to run in a containerised environment.

OpenAPI Specification

A standard, programming language-agnostic interface description for REST APIs. REST APIs allow both humans and computers to discover and understand the capabilities of a service without requiring access to source code, additional documentation, or inspection of network traffic. When properly defined via OpenAPI, a consumer can understand and interact with the remote service with a minimal amount of implementation logic.

In this Solution Pattern you follow a design-first approach, based on the OpenAPI specification, to define a RESTful API that accepts a name parameter and echoes it back in the response body. You then use the resulting Open API definition to generate the code for your API and deploy it in a containerised environment.

Following a design first approach to API development is self-documenting, and provides a clear API definition for partners and end-users of your API. It also allows you to identify important design decisions prior to coding and deployment.

Creating the Slack Connection in Fuse Online

  1. Log in to the Red Hat Fuse Online console.

    Note

  2. Choose Connections from the left navigation menu.

  3. Click the Create Connection button in the top right corner.

  4. Choose Slack from the list of displayed Connectors. You are prompted for configuration details.

  5. Obtain a URL for the Slack Webhook URL field:

    1. Navigate to the Slack Apps portal at api.slack.com/apps.

    2. Click Create new App.

    3. Enter the following in the App Name field:

      Fuse Online Demo

    4. Choose low-code-workspace from the Development Slack Workspace menu.

    5. Click "Create App".

    6. On the following screen under the Add features and functionality click Incoming Webhooks.

    7. Change the Activate Incoming Webhooks switch to the On position.

    8. Scroll down and click Add New Webhook to Workspace.

    9. In the Post to field select low-code-demo and click Allow.

      You are redirected back to the Incoming Webhooks page.

    10. Copy the Webhook URL displayed at the bottom of the page using the Copy button.

  6. Paste the copied Webhook URL value into the Slack Webhook URL field in Fuse Online.

  7. Obtain a value for the Token for Accessing Slack API field:

    1. Navigate to the Slack API Token portal at api.slack.com/custom-integrations/legacy-tokens.

    2. Scroll down to the Legacy token generator section.

    3. Click Create token button next to low-code-workspace.

    4. Copy the generated token.

  8. Paste the generated token into the Token for Accessing Slack API field in Fuse Online.

  9. Enter the following in the Sending User Name for Messages field:

    Fuse Online

  10. Click Validate in Fuse Online and verify that a Slack has been successfully validated. message appears. This validation sends a message to the low-code-demo channel.

  11. Click Next.

  12. Enter the following in the Connection Name field on the Add Connection Details screen:

Creating an API Integration in Fuse Online

Creating an Integration with API details

In this procedure, you create an integration with an API and include a data type. A Data Type represents a data structure that can be passed to your API and can then be used as parameters in your API definition and within the Fuse Online Integration Flow Editor to transform and map data as part of an Integration.

  1. Log in to the Red Hat Fuse Online console.

  2. Choose Integrations from the left navigation menu.

  3. Click the Create Integration button to start the New Integration wizard.

  4. Choose API Provider on the subsequent Choose a Start Connection screen.

  5. When prompted choose Create and click Next to navigate to the API Designer.

  6. Rename your API from "Untitled API" to:

    Greeting API

  7. Edit the Description:

    My greeting API

  8. Click Add a data type under the Data Types heading on the left of the API Designer screen.

  9. In the Enter Basic Information section enter the following in the Name field:

    Name

  10. Enter the following JSON in the Enter JSON Example field:

    {
    "name": "shadowman"
}

  11. Scroll down and click Save.

Is </> Name listed under the Data Types on the API Designer screen?

Verify that you followed each step in the procedure above. If you are still having issues, contact your administrator.

Creating a POST Resource Path

A Path represents an API endpoint/operation and the associated parameters required to invoke it.

  1. Click Add a path under the Paths heading on the left of the API Designer screen.

  2. In the window that appears, enter the following in the Path field:

    /greeting

  3. Click Add to confirm your entry.

  4. Click your new /greeting endpoint under the Paths heading.

  5. Click Add Operation under the POST icon in the Operations section on the right.

  6. Enter the following in the Summary field:

    Greet with name

  7. Enter the following in the Operation ID field:

    greetname

  8. In the Request Body section, click Add a request body.

  9. Choose Name as the type.

  10. In the Responses section click Add response.

    1. In the window that appears choose 200 OK as the option.

    2. Click Add.

    3. Click No description beside the 200 OK response and enter the following in the Description field:

      Greeting response

    4. Choose String as the type. Fuse Online automatically completes the type String as String.

  11. Click Save in the top right corner of the page to save your work. The New Integration wizard is displayed.

  12. Click Next.

  13. Click Publish on the New Integration wizard screen.

  14. When prompted to Give this integration a name:

    1. Enter the following in the Integration Name field:

      Greeting Integration

    2. Enter the following in the Description field:

      my greeting api

  15. Click Save and Publish to save your API design.

Is a Greeting Integration entry listed in the Fuse Online Home screen?

Implementing and publishing the API

  1. Log in to the Red Hat Fuse Online console.

  2. Select Integrations from the left hand menu.

  3. Choose Edit from the menu for the Greeting Integration entry. The Operations page appears.

  4. Click the Create flow button. The Add to Integration page appears displaying the Provided API and the Provided API Return Path.

  5. Click the blue plus icon to add a step. The left hand side of the Choose a connection page lists the steps in your Integration.

  6. Select your Greeting Slack Target.

  7. When prompted to Choose an action select Channel. You can use this to send a message to a specific channel in your Slack workspace.

  8. Use the Channel menu to select the low-code-demo channel and click Next. The Integration Flow Editor should now list your Slack connection with a Data Type Mismatch warning.

  9. Click the Warning Icon and choose Add a data mapping step.

  10. From the Data Mapper screen click the body field in the Source panel to expand it.

  11. Click the name field under the body, then click the message field in the Target panel. This maps the value of the incoming HTTP request body to the outgoing Slack message property.

  12. In the right hand Mapping Details panel, click the Add Transformation under the Targets section to add a transformation.

  13. Change the transformation type from Append to Prepend.

  14. Enter the following in the string field under the menu:

    Hello from

  15. Click Done to return to the Integration Flow Editor. The Integration Flow Editor should now list your Provided API Return Path with a Data Type Mismatch warning.

  16. Click the Warning Icon and choose Add a data mapping step.

  17. From the Data Mapper screen click the Request field in the Source panel to expand it.

  18. Click the name field under the body, then click the message field in the Target panel. This maps the name value from the incoming HTTP request body to the body of the outgoing Slack message.

  19. Click Done to return to the Integration Flow Editor.

  20. Click Publish.

  21. Click Save and Publish to trigger an Source to Image (S2I) build.

  22. Wait until Fuse Online reports your deployment was successful.

  7. In the MAPPING RULES section, add a POST mapping with the pattern /greeting.

    1. Select Update & test in Staging Environment
